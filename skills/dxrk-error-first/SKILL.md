# 🔥 DXRK Error-First Protocol

> "Si hay errores, corrígelos primero. Siempre."

---

## Regla Fundamental

```
┌─────────────────────────────────────────────────────────────┐
│  🚨 SI DETECTAS UN ERROR O FALLO:                          │
│     1. CORRIGE EL ERROR PRIMERO                            │
│     2. VERIFICA QUE ESTÁ CORREGIDO                         │
│     3. LUEGO CONTINÚA CON EL PROYECTO                      │
└─────────────────────────────────────────────────────────────┘
```

---

## Checklist de Errores a Verificar

Antes de entregar o avanzar, siempre verificar:

- [ ] **Sintaxis** del código (HTML, JS, Python, etc.)
- [ ] **Tests passing** (si hay tests)
- [ ] **Imports** correctos
- [ ] **Paths** existen
- [ ] **Dependencies** instaladas
- [ ] **Build** exitoso (si aplica)
- **Si algo falla → CORREGIR PRIMERO → Verificar → Continuar**

---

## Comportamiento

1. **Al iniciar tarea nueva**: Verificar que el entorno esté limpio
2. **Al hacer cambios**: Verificar que no rompiste nada
3. **Al terminar**: Siempre dejar todo funcionando

---

## Para recordármelo

Cuando estés trabajando en algo y detectes un error,decime:
